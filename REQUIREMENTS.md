# 项目背景与需求说明

> 本文件交代项目的来龙去脉与核心需求，所有 Agent 在动手前请先阅读本文件，再读 [PROJECT_PLAN.md](PROJECT_PLAN.md)。

---

## 一、项目由来

项目发起人需要为自己孩子取名，在与 AI 多轮对话中发现：

1. **市面上的取名网站普遍不专业**：要么只算简体笔画（错误），要么"缺啥补啥"（错误），要么字义来源是套话（无典籍可溯）。
2. **不同 AI 给出的名字评分相互矛盾**：同一个名字，A 模型打 95 分，B 模型一查数理发现人格大凶。
3. **传统取名学的真正门槛在数据**：康熙笔画、八十一数理、典籍出处、名人字库——这些是规则引擎的基础，没有结构化字库做不出准确判断。

因此发起人决定：**自己做一套真正严谨、可解释、可溯源的智能取名系统**。

---

## 二、项目目标

打造一个综合**中国传统命理 + 文化典籍 + 现代 AI 复审**的取名系统，输出的每个名字都做到：

| 维度 | 要求 |
|---|---|
| 命理合理 | 符合孩子八字用神（不是缺啥补啥，是看调候/扶抑） |
| 数理吉利 | 三才五格按康熙繁体计算，避开 19/20 等大凶数 |
| 文化有据 | 字必须能在典籍或名人名字中找到出处 |
| 意境优美 | 字义画面感强，无生僻字、无负面引申 |
| 好听好写 | 普通话+方言谐音排查，笔画结构均衡 |
| 可解释 | 每个名字附五维评分卡 + 详细溯源，不是黑盒 |

## 三、核心需求清单（来自项目发起人原话）

### 必备功能 ✅

- [ ] **N1**：输入孩子生辰（公历/农历 + 时辰 + 出生地），自动排八字、定用神
- [ ] **N2**：支持指定**必含字**（例如女孩名要含"雯"），并指定位置（首字/末字/任意）
- [ ] **N3**：候选名字中的字必须能从**名人名字**或**典籍**中找到出处，并明确展示来源
- [ ] **N4**：按**五维评分体系**给名字打分并排序：
  - 八字补益 30%
  - 三才五格 25%（康熙繁体笔画）
  - 字义寓意 20%
  - 音律读音 15%
  - 字形书写 10%
- [ ] **N5**：每个推荐名字提供**详细评分卡**，五维拆解、可解释
- [ ] **N6**：支持父母八字辅助参考（与孩子八字的匹配度，可选项）

### 期望功能 ⭐

- [ ] **N7**：支持指定**必避字**（如长辈名讳、不喜欢的字）
- [ ] **N8**：支持指定**风格偏好**（古典文雅 / 大气磅礴 / 温柔婉约 / 现代简约 等）
- [ ] **N9**：支持指定**来源偏好**（诗经 / 楚辞 / 唐诗 / 宋词 / 论语 / 周易 / 现代名人 / 明星）
- [ ] **N10**：用户可自定义**评分权重**（默认权重 vs 高级自定义）
- [ ] **N11**：LLM 复审层，做谐音、时代感、性别气质的最终筛查
- [ ] **N12**：导出 PDF 评分报告

### 加分功能 🌟

- [ ] **N13**：易经卦象辅证（基于出生时辰起卦，建议契合卦意的字）
- [ ] **N14**：改名场景支持（输入现名，分析问题 + 给微调建议）
- [ ] **N15**：批量对比（用户上传候选名单，系统重排序）
- [ ] **N16**：方言谐音库（粤/吴/闽/川各地）

## 四、典型用户场景

### 场景 A · 全新取名（最主线）

```
张先生的孩子 2026 年某月某日某时出生，男孩。
打开系统 → 输入生辰、姓氏、性别
        ↓
系统排八字：日主甲木，丑月生，用神火土
        ↓
系统输出 Top 10：
  1. 张维城（木→土→火，五格全吉，《诗经》"宗子维城"）
  2. 张敦翔（木→火→火，五格全吉，敦厚翱翔）
  ...
每个名字点开 → 五维评分卡 + 名人参照 + 典籍出处
```

### 场景 B · 指定必含字

```
李女士想给女儿取名，必须含"雯"字（家族传统/个人喜好）。
打开系统 → 输入生辰、姓氏、性别、必含字"雯"、位置"末字"
        ↓
系统在所有"X+雯"组合里筛出五格全吉的笔画
        ↓
按八字、字义、典籍来源、名人参照排序
        ↓
Top 10：张诗雯、张芷雯、张紫雯...
        ↓
每个名字展示："诗雯"中"诗"出自《毛诗序》，名人有"刘诗雯（乒乓世冠）"
```

### 场景 C · 父母期望对比

```
父母心仪 3 个名字：张景尧、张珂玮、张景宸
        ↓
系统对每个名字做完整评分
        ↓
返回横向对比表，指出"张景宸"地格 22 凶等具体问题
        ↓
帮助父母做最终决策
```

## 五、不做的事（项目边界）

为了聚焦，以下功能**暂不纳入**：

- ❌ 算命占卜（仅取名相关的命理）
- ❌ 风水堪舆
- ❌ 公司起名 / 品牌起名（仅个人取名）
- ❌ 英文名
- ❌ 改运改名玄学营销

## 六、关键设计原则

1. **专家规则 > LLM 自由发挥**：核心评分用规则引擎，LLM 只做复审/解释。
2. **数据可溯源**：每条字义、每个典籍出处、每位同字名人都要有具体来源，不杜撰。
3. **康熙繁体笔画**：所有数理计算用康熙繁体，不用简体（简体是常见错误）。
4. **用神不是缺啥补啥**：严格按调候 → 扶抑 → 通关 → 病药四步走。
5. **黑盒不可接受**：评分必须可拆解、可解释，用户能看到每一分的来由。
6. **多 Agent 协作友好**：每个任务原子化，文档化进度，便于 Codex 等其他 Agent 接力。

## 六·甲、取名核心理念（重中之重）

> ⚠️ **本节是项目的灵魂，所有 Agent 在写评分逻辑前必读。**

### 一句话总纲

> **八字定方向，字义定气质，五格守底线。**

### 五维真实优先级

```
┌──────────────────────────────────────────────┐
│  ① 八字用神  ★★★★★  灵魂 · 决定方向          │
│  ② 字义意境  ★★★★☆  气质 · 决定格调           │
│  ③ 音律字形  ★★★★   日常 · 决定使用体验       │
│  ④ 三才五格  ★★★    避坑 · 决定下限           │
│  ⑤ 易经卦象  ★★     辅证 · 锦上添花           │
└──────────────────────────────────────────────┘
```

### 为什么八字用神才是核心

1. **名字的本质功能是"补气"**：人的八字一出生就定了，先天五行有缺、有偏、有寒、有燥。名字是孩子终身佩戴的"五行护身符"，作用是**调和、补益、流通**这个先天结构。这才是名字"改运"的真正机制。
2. **用神错了，五格再吉也没用**：
   - 反例：某甲木冬生宝宝，用神为火。"张沛霖"（沛+霖都是水）五格全大吉、三才相生——但加重水寒，木更冻，**这是"数理满分的坏名"**。
   - 正例："张昕烨"（昕+烨都是火）五格有小瑕疵——但直接补丙火，**这是"数理及格的好名"**。
   - **传统派会毫不犹豫选后者。**

### 为什么三才五格不是最重要

| 维度 | 八字命理 | 三才五格 |
|---|---|---|
| 起源 | 唐代李虚中 → 宋代徐子平 | 日本熊崎健翁，1918 年 |
| 历史 | 1000+ 年 | < 100 年 |
| 看孩子 | 看（生辰精准定位） | 不看（只算姓名笔画） |
| 性别 | 有区分 | 几乎无区分 |
| 文化 | 汉字本义/字源 | 简化为数字加减 |

**结论**：三才五格的作用是"**别让名字给孩子拖后腿**"，不是"让名字给孩子加分"。**保下限，不能定上限。**

### 关键避雷区（市面常见误区）

- ❌ **"五格全吉就是好名"** → 数理是必要条件，不是充分条件
- ❌ **"缺啥补啥"** → 缺的五行可能正是忌神，补了反而坏；必须先定用神
- ❌ **"网站给我打 99 分"** → 真正好的评分必须拆解到每一维，黑盒打分不可信
- ❌ **"三才五格 = 中国传统"** → 是日本近代发明，不是中华传统命理核心
- ❌ **"为了五格凶吉牺牲字义"** → 字义平庸却数理满分，远不如字义优美数理及格

### 本项目的权重决策与折中

> 项目最终权重（见上文项目目标章节）：
> 八字 30% / 五格 25% / 字义 20% / 音律 15% / 字形 10%

**严格按传统命理派应该这样配**：

```
八字补益    40-50%  ← 真正的核心
字义意境    25-30%  ← 紧随其后
音律字形    15%
三才五格    10-15%  ← 只做避坑
```

**为什么我们给五格 25%（偏高）？折中考虑**：

1. 五格容易量化，规则清晰，评分稳定
2. 八字用神判定有"流派分歧"，需要 LLM + 专家辅助
3. 给五格更高权重 → 系统输出更稳定可解释
4. **但不会让五格凌驾于八字之上**（八字仍然是单维最高分项）

### 严格执行的取名顺序 🔑

> **所有 Agent 写算法时必须严格按此顺序，不可颠倒：**

```
1. 八字排盘   →  孩子是谁
2. 定用神     →  需要补什么
3. 选五行     →  字库筛选范围
4. 笔画穷举   →  五格全吉的组合
5. 字义筛选   →  典籍出处 + 名人参照
6. 音形复审   →  谐音、字形、性别气质
7. LLM 终审   →  时代感、整体韵味
```

❌ **错误顺序**：先穷举五格全吉组合 → 再回头看八字符不符合
✅ **正确顺序**：先定用神五行 → 在符合的五行字里穷举吉数组合

### 三句话铁律

> **1. 名字是给孩子的，不是给五格表的。**
> 一切以孩子的八字为本，名字是为这个孩子量身定做的护身符。
>
> **2. 五格是"别太烂"，八字是"要真好"。**
> 五格保下限，八字决定上限。
>
> **3. 文化是名字的灵魂，不要为了五行牺牲字义。**
> 如果一个字五行完美但字义平庸，远不如一个字义有典故、意境优美、五行也对路的字。

## 七、技术栈（已决策）

| 层 | 选型 | 理由 |
|---|---|---|
| 后端 | Python + FastAPI | 命理库生态在 Python |
| 前端 | Next.js + TypeScript + Tailwind + shadcn/ui | 评分卡可视化需求 |

| 命理库 | lunar-python / sxtwl | 农历、节气、八字 |
| 部署 | Vercel (前端) + Railway (后端) | 小流量起步 |

## 八、成功标准（验收）

MVP 上线后达到以下指标才算成功：

- [ ] **准确性**：Top 100 案例经资深取名师审核，准确率 ≥ 90%
- [ ] **响应速度**：单次推荐生成 < 5 秒（含 LLM 复审）
- [ ] **可解释性**：每个推荐名字都附完整五维评分卡 + 溯源
- [ ] **用户满意度**：5+ 真实用户测试，满意度 ≥ 4/5
- [ ] **数据规模**：字库 ≥ 5000 字、名人库 ≥ 1 万条、典籍 ≥ 8 本
